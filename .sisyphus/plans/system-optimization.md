# Vermilion Bird 系统优化计划

## 优化项概览

### 高优先级

1. **Calculator 技能安全重构**
   - 问题：使用 `eval()` 存在安全隐患
   - 方案：使用 `ast` 模块安全解析表达式
   - 预期：完全移除 `eval()` 调用

2. **统一服务管理**
   - 问题：调度器等服务启动逻辑分散
   - 方案：创建 `ServiceManager` 统一管理
   - 预期：所有服务通过统一接口管理

### 中优先级

3. **完善类型注解**
   - 问题：377 个 Pyright 诊断错误
   - 方案：分批次修复类型问题
   - 预期：错误减少 90% 以上

4. **增强配置验证**
   - 问题：配置加载后验证不足
   - 方案：添加 Pydantic 验证器
   - 预期：无效配置在启动时就报错

5. **统一异常处理**
   - 问题：各模块错误处理不统一
   - 方案：定义统一异常基类和子类
   - 预期：异常处理一致且可追踪

### 低优先级

6. **添加健康检查**
   - 问题：无法监控服务状态
   - 方案：创建健康检查模块和 CLI 命令
   - 预期：可一键检查所有服务健康状态

7. **飞书适配器重试机制**
   - 问题：API 调用失败无重试
   - 方案：使用现有重试装饰器
   - 预期：飞书 API 调用自动重试
